# ルーティンノート v115 リリース前監査

実施日: 2026-07-20

対象: GitHub Pages版PWA（iPhone Safariを主対象）

## 結論

- **少人数のテスター投入: 条件付きGO**。下記P0のiPhone実機確認を終えれば、3〜5人程度のクローズドテストを始められる状態。
- **不特定多数への公開: まだNO-GO**。動画・音源を含む完全バックアップがなく、フィードバックの集約先も未設定のため。
- 今回は大規模なファイル分割をせず、データ保全と実行時不具合を優先してv115へ修正した。

## 今回確認・修正した内容

### データ保全

- 練習日をUTCではなく端末の現地日付で保存。日本時間0〜8時台が前日になる不具合を修正。
- 保存予約中にアプリがバックグラウンドへ移った場合、即時保存するよう変更。
- IndexedDBとlocalStorageの両方に保存時刻を持たせ、書き込み中断後は新しい方を採用。
- 保存先が利用できない場合、一度だけユーザーへ空き容量確認を表示。
- JSON復元時にデータ構造とIDを検証し、旧バックアップには移行処理を適用。

### 実行時の安定性

- 画面遷移前に開始した音源ロードが、遷移後の別画面へ反映される競合を防止。
- 動画・音源トリム画面を閉じた際に、メディアとObject URLを解放。
- 音源・動画の長さ取得にタイムアウトと後始末を追加。
- 再生失敗を未処理Promiseにせず、ユーザーへ通知。
- 起動失敗時にローディング画面で停止せず、再読み込み案内を表示。
- Service Workerが画像・音源の取得失敗時にHTMLを返す挙動を廃止。成功した200応答だけを保存。

### 機能抜け・アクセシビリティ

- パート練習へ0.5× / 0.75× / 1.0× / 1.25×の再生速度を追加。ルーティンごとに保持。
- 画面拡大禁止を解除。
- PWAの背景色・テーマ色を現在のルーズリーフデザインへ統一。

### 軽量化・保守性

- UIから到達不能だった旧「構成ビルダー」を削除し、編集画面のタイムライン機能へ一本化。
- `scripts/release-check.mjs`を追加。構文、版番号同期、Service Worker対象ファイル、画面拡大、旧コード残存、主要ファイル容量を一括確認できる。
- v115の主要コードgzip概算は約108KiB。背景SVGを加えても初期転送は概ね123KiB相当。サンプル音源約7.4MBと動画は初回表示では取得せず、選択時だけ読み込む。

## 軽量化の次の候補

### 優先度A: テスター後に検討

1. **英語辞書の遅延読み込み**

   日本語利用時は`i18n.js`を読み込まない構成にすると、公開版実測で約11KBの転送を削減できる。ただし言語切替時の追加ロードとオフラインキャッシュ設計が必要。

2. **画面単位のJavaScript分割**

   技動画・録音・分析を別モジュールに分ければ初期解析量を減らせる。現状の初期転送量はすでに小さく、テスター前に行うと回帰リスクの方が大きいため見送った。

3. **サンプルの追加方法を選択制にする**

   9本の動画と楽曲を一括追加すると端末内でキャッシュとIndexedDBの両方に持つ場合がある。テスターの用途に合わせ、動画なしの軽量サンプルを用意すると保存容量を抑えられる。

### 優先度B: 性能より保守性

- `app.js`のインライン`onclick`を画面単位のイベント委譲へ置換し、自動テストしやすくする。
- 状態スキーマの版番号と段階別マイグレーションを明文化する。
- CSSの画面別整理は保守性には効くが、GitHub Pagesが圧縮配信しているため体感速度への効果は小さい。

## テスター投入前のP0チェック

### iPhone実機（最低1台、できれば新旧2台）

- Safariで公開URLを開く → ホーム画面に追加 → PWAとして再起動。
- ルーティン作成 → 即座にアプリを閉じる → 再起動後も最後の編集が残る。
- 通し練習: 5秒カウント、楽曲自動再生、クリーン、失敗、中止、セッション終了。
- パート練習: A/Bドラッグ、ループ間隔、4段階の再生速度、バックグラウンド復帰。
- カメラ撮影: 20秒自動停止、保存、トリム、ルーティンへの紐づけ。
- 写真アプリ由来の動画: 480p/360p圧縮、画面ロックや着信後に壊れないこと。
- マイク録音: 許可、録音、停止、再生、削除。通し画面を離れた場合も保存されること。
- 機内モードで再起動し、アプリ本体と一度取得済みの素材が開けること。
- 日本語/英語切替、VoiceOver、文字拡大、ピンチズーム。

### データ復旧

- v112のJSONをv115へ復元し、ルーティン・分析・設定が保たれること。
- 復元直後に「音源・動画はJSONに含まれない」ことをテスターへ明示する。
- 端末の空き容量が少ない場合の動画保存失敗を確認する。

## 残るリリース課題

### P0: 公開前に必要

1. **完全バックアップ方針**

   現状のJSONはルーティン・記録・設定のみで、動画・楽曲・録音Blobを含まない。クローズドテストでは「元ファイルを残す」運用で開始できるが、一般公開前にはZIP等の完全書き出し、またはクラウド同期のどちらかを決める。

2. **実機メディア検証**

   カメラ、マイク、動画圧縮はデスクトッププレビューだけでは合格にできない。iPhone Safariとホーム画面追加後の両方で確認する。

3. **フィードバック集約**

   `FEEDBACK_ENDPOINT`は空で、現在はメールアプリへの引き継ぎ。少人数テストは可能だが、人数を増やす前にGAS等の集約先を設定し、送信成功・失敗を実機確認する。

### P1: テスト期間中に詰める

- Service Worker更新時に「新しい版があります」と案内する更新UI。
- 使用容量の表示と、不要な孤立Blobを掃除するメンテナンス処理。
- シートのフォーカストラップ、Escape操作、VoiceOver読み上げ順。
- Safari以外（Android Chrome）のカメラ形式・MediaRecorder形式確認。
- テスター用の1分使い方動画、既知の制約、フィードバック手順を1ページにまとめる。

### P2: 一般公開・ストア化を考える段階

- プライバシー説明、利用上の注意、問い合わせ先、データ削除方法の公開ページ。
- クラッシュ収集を導入する場合の同意・匿名化・保存期間。
- ブラウザ保存だけで運用するか、アカウント同期を持つかの製品判断。
- 多言語追加時の翻訳レビューと、フランス語の長文・中国語フォントの実機確認。

## 毎回のリリース確認コマンド

```bash
node scripts/release-check.mjs
node --check app.js
node --check i18n.js
node --check sw.js
git diff --check
```

ビルド成功、ブラウザ確認、iPhone実機確認、GitHub Pages反映は別々の合格項目として記録する。
