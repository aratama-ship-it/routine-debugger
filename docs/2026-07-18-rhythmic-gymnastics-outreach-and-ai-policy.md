## 二種類のリスクを分ける

### 1. 技そのもののリスク

その技を単独で実施した時の難度、失敗しやすさ、怪我の可能性など。現状の `risk` に近いが、難度と安全上の危険性は
本来別概念なので、将来的には混同しない。

### 2. 演技全体への影響リスク

その技を構成へ入れたことで、演技全体の成功、流れ、回復、後続の技へどの程度影響し得るかという観点。

候補となる要素:

- 失敗すると演技が中断するか、すぐ回収できるか。
- 道具、隊形、立ち位置、向きが崩れ、後続へ影響するか。
- 予定時間や音楽との同期を失うか。
- 直後の技・移行で失敗が増える傾向があるか。
- 団体演技の場合、他の演者へ崩れが波及するか。
- 成功しても疲労、視線、距離、持ち替えなどが次の技へ影響するか。

この値をAIが単一スコアとして断定しない。まずは次のように観測可能な情報を分けて記録する。

```text
技の結果       : 成功 / 乱れたが回収 / 失敗して回収 / 中断 / 安全上の回避
直後への影響   : なし / あり / 不明
影響範囲       : 次の技 / 複数の後続技 / 演技全体 / 他の演者
回復所要       : すぐ / 数ステップ / 回復できず
本人の仮説     : 疲労、位置、タイミング、視線、道具、心理など（任意）
```

表示する時は「全体影響リスク 78」のような根拠不明の点数より、
「この技の乱れ5回中、4回で次の2技以内にも乱れを記録」のように分子・分母と観測範囲を示す。
固定順の演技では原因を分離できないため、**影響が観測されたこと**と**この技が原因であること**を区別する。

## AI分析の利用方針

AI APIを、通し練習のたびに自動実行する前提にはしない。

### 常時行ってよいこと（AI APIなし）

- 記録の保存
- 到達回数、成功・回収・中断の件数集計
- 技・移行・演技バージョン別の基本集計
- 前回メモと「次回試すこと」の表示
- 端末内で計算できる既知ルールによる注意表示

### AI APIを使う候補

- ユーザーが「分析する」を明示的に押した時
- 一定数の練習記録が溜まり、振り返りたい時
- 本番前、構成変更前後、問題箇所が分からない時
- 指導者や選手へ共有する要約を作る時

### 利用上限の第一候補

- **1ユーザーにつき1日1回まで**を無料枠の仮案とする。
- その日の最新データをまとめて一度に送り、同じ記録への重複分析を避ける。
- 分析結果と対象データの版を保存し、再表示ではAPIを呼ばない。
- 記録が少なすぎる場合はAPIを使わず、「まだ比較できない」と端末内で返す。
- 上限は原価、継続率、実際の利用頻度を測ってから変更する。現時点では確定仕様にしない。

### AIが返すもの

1. 観測された偏り
2. 考えられる複数の解釈
3. 次の練習で試せる小さな比較
4. 判断に不足している記録

原因の断定、自動的な安全判定、無根拠な練習処方は返さない。AI分析はコア機能ではなく、端末内の記録を必要時に
読み解く補助層として扱う。
